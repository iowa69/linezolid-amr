1 · Overview & scientific rationale

linezolid-amr runs two complementary analyses from a single command and combines them into one report:

  1. AMRFinderPlus (NCBI) on the assembly — detects acquired AMR genes (cfr, optrA, poxtA, …) and known point mutations (ribosomal proteins L3, L4, L22; canonical 23S sites).
  2. 23S rRNA mutation-frequency profiling on the raw reads — maps the reads to a species-specific 23S reference and reports allele frequencies at canonical linezolid-resistance positions (G2576T, G2505A, G2447T, C2534T, A2503G, …).
Why the read-level step matters. Most clinically important Gram-positive pathogens carry multiple copies of the 23S rRNA operon (4–6 in S. aureus and Enterococcus spp.; 4 in S. pneumoniae; M. tuberculosis is the exception with a single copy). Linezolid resistance can be heteroresistant: a G2576T mutation may be present in only 1–3 of the operon copies, putting the population-level allele frequency at 15–60 %. The assembly consensus base remains G, so assembly-only tools miss the resistance. Reading allele frequencies directly off the raw reads is the only reliable way to detect this.

The same --organism flag drives both AMRFinderPlus species mode and 23S reference selection. Mutations are always reported in E. coli K-12 23S numbering — the clinical literature standard — via a pairwise-alignment-derived position map that is computed once when references are fetched.

2 · Installation

Option A · From bioconda recommended once published

conda create -n linezolid-amr -c bioconda -c conda-forge linezolid-amr
conda activate linezolid-amr

Bioconda installs all binary dependencies automatically: ncbi-amrfinderplus, minimap2, samtools, bcftools, plus the Python libraries click, pysam, biopython, pandas.

Option B · From source (development)

git clone https://github.com/sintetico87/linezolid-amr
cd linezolid-amr
conda create -n linezolid-amr -c bioconda -c conda-forge \
    python=3.11 ncbi-amrfinderplus minimap2 samtools bcftools
conda activate linezolid-amr
pip install -e .

Verify the installation

linezolid-amr --version
# linezolid-amr, version 0.1.0

linezolid-amr list-organisms

# Enterococcus_faecium      Enterococcus faecium DO 23S rRNA (first of 6 copies)
# Mycobacterium_tuberculosis Mycobacterium tuberculosis H37Rv rrl (single 23S rRNA copy)
# Staphylococcus_aureus     Staphylococcus aureus NCTC 8325 23S rRNA (Sa23SA, first of 5 copies)
# Streptococcus_pneumoniae  Streptococcus pneumoniae TIGR4 23S rRNA (first of 4 copies)

3 · One-time setup

3.1 · AMRFinderPlus database (~150 MB)

amrfinder -u

This downloads the latest curated AMR gene/mutation database from NCBI into the AMRFinderPlus install path. Re-run whenever you want to refresh.

3.2 · 23S references (~25 KB per species)

# Fetch references for the species you analyze
linezolid-amr fetch-references --organism Staphylococcus_aureus

# Or pull all five at once
linezolid-amr fetch-references

# Optional — speed up NCBI requests with your API key
export NCBI_API_KEY=<your-key>
linezolid-amr fetch-references

For each organism, fetch-references:

  1. Downloads the exact 23S rRNA slice from NCBI E-utilities (verified genome coordinates).
  2. Downloads the E. coli K-12 rrlB 23S as a coordinate master.
  3. Performs a global pairwise alignment to build a per-base species-position ↔ E. coli-position map.
  4. Writes a BED file with canonical LZD-resistance positions translated into species coordinates.

The cache lives at:

$XDG_DATA_HOME/linezolid-amr/references/
# or, by default:
~/.local/share/linezolid-amr/references/

# Override the location:
export LINEZOLID_AMR_REFDIR=/path/to/refs

Cache contents per species:

FilePurpose
Staphylococcus_aureus_23S.fastamapping reference (FASTA + .fai)
Staphylococcus_aureus_23S_lzd_positions.bedcanonical LZD sites in species coords
Staphylococcus_aureus_23S_position_map.tsvper-base E. coli ↔ species position map
ecoli_K12_23S_rrlB.fastashared coordinate master (downloaded once)

4 · Quick start

linezolid-amr run \
    --assembly  sample.fasta \
    --r1        sample_R1.fastq.gz \
    --r2        sample_R2.fastq.gz \
    --organism  Staphylococcus_aureus \
    --outdir    results/sample \
    --sample    sample \
    --threads   8

That's the whole pipeline. On a typical 200 MB paired-end run for S. aureus, end-to-end time on 8 threads is ~2 minutes (mostly read mapping).

Useful flags

FlagDefaultMeaning
--min-af0.01min alt-allele frequency to report (1 %)
--min-depth20min reads at a position to make a resistance call
--skip-amrfinder—skip the AMRFinderPlus step
--skip-rrna23s—skip the 23S analysis step
--threads4threads forwarded to amrfinder / minimap2 / samtools / bcftools

6 · Output directory structure

After a successful linezolid-amr run the output directory has this layout:

results/sample/ ├── amrfinder/ │ ├── amrfinder.tsv # AMRFinderPlus raw TSV (all hits) │ └── amrfinder.log ├── rrna23s/ │ ├── sample.23S.bam # sorted, indexed alignment │ ├── sample.23S.bam.bai │ ├── sample.23S.vcf.gz # all variants in 23S (bcftools, ploidy=1) │ ├── sample.23S.vcf.gz.csi │ ├── sample.23S_lzd_pileup.tsv # allele counts/freqs at canonical LZD positions │ ├── sample.23S.map.log │ └── sample.23S.vcf.log ├── sample.linezolid_amr.json # full machine-readable report └── sample.linezolid_amr.txt # human-readable summary

7 · Output file formats

7.1 · amrfinder.tsv

The standard AMRFinderPlus TSV — one row per hit. Key columns: Gene symbol, Sequence name, Class, Subclass, % Coverage, % Identity, Method, Closest reference accession. Hits relevant to linezolid (cfr / optrA / poxtA / 23S / L3 / L4 / L22) are flagged in the combined JSON report.

7.2 · sample.23S_lzd_pileup.tsv

ColumnMeaning
organismorganism passed via --organism
ref_contigreference 23S contig name
species_position1-based position in the species 23S reference
ecoli_positioncorresponding E. coli K-12 23S position
ref_basewild-type base
depthtotal reads at this position (after BAQ/min-base-quality filter)
countsper-base counts (e.g. A=2;G=820)
alt_allelesnon-ref alleles passing thresholds: base:count:af[*=resistance]
is_resistanceTrue if a known resistance allele was observed above thresholds
drugresistance phenotype the position is associated with

7.3 · sample.23S.vcf.gz

Full bcftools VCF of all variants found within the 23S locus (ploidy=1, depth up to 100 000). Use this to discover novel mutations that are not in the canonical resistance list. Headers include AD and DP tags for allele depth and total depth.

7.4 · sample.linezolid_amr.json

The combined machine-readable report. Top-level structure:

{
  "tool": "linezolid-amr",
  "version": "0.1.0",
  "sample": "...",
  "organism": "Staphylococcus_aureus",
  "parameters": { ... },
  "summary": {
    "linezolid_resistance_call": false,
    "n_total_amr_hits": 4,
    "n_linezolid_amr_hits": 0,
    "n_23s_positions_evaluated": 11,
    "n_23s_positions_with_resistance_allele": 0,
    "linezolid_amr_genes_detected": [],
    "linezolid_23s_positions": []
  },
  "amr_hits": [ ... 1 dict per AMRFinder hit ],
  "rrna23s_pileup": [ ... 1 dict per canonical position ],
  "rrna23s_vcf": "results/.../sample.23S.vcf.gz"
}

7.5 · sample.linezolid_amr.txt

Human-readable plain-text summary suitable for emailing or pasting into a lab notebook.

8 · Interpreting results — worked example

Reading the result. Mapping all reads to a single 23S copy pools coverage from all five operons, yielding ~750–840× depth. Every canonical position is >99 % reference base. The few alt reads (e.g. T=3 at position 2603) sit around 0.4 %, below the 1 % AF threshold — consistent with Illumina sequencing noise. Combined call: linezolid-susceptible.

What a resistant sample would look like

A G2576T heteroresistant isolate where the mutation is fixed in 2/5 operons would typically show:

ecoli_position  ref_base  depth  counts            alt_alleles            is_resistance
2576            G         800    G=480;T=320       T:320:0.4000*          True

(The trailing * marks the alt allele as a known resistance base.)

9 · Heteroresistance & thresholds

The interpretation of an alt-allele frequency at a 23S position depends on the organism's number of rRNA operons:

OrganismOperons1 mutated copy2 mutated3 mutated
S. aureus5~20 %~40 %~60 %
E. faecalis4~25 %~50 %~75 %
E. faecium6~17 %~33 %~50 %
S. pneumoniae4~25 %~50 %~75 %
M. tuberculosis1——100 % only
Thresholds. The defaults (--min-af 0.01, --min-depth 20) are chosen so that: (a) Illumina ~0.5 % per-base error stays below the AF floor at typical depths; (b) clinically meaningful heteroresistance (≥1 operon out of 4–6) is always reported. Lower --min-af only if you have very deep sequencing and a careful manual review plan.

10 · Troubleshooting

SymptomLikely cause & fix
amrfinder failed (exit 1): "No valid AMRFinder database is found" Run amrfinder -u once. The DB lives outside the conda env data dir.
References for 'X' not found in cache Run linezolid-amr fetch-references --organism X. Requires internet.
NCBI E-utils timeout / 429 Set NCBI_API_KEY in the environment. Retry — the fetcher already retries 3×.
23S coverage is much lower than expected (<50×) Wrong organism. Verify with a taxonomic profiler (Kraken2, etc.) and re-run.
"organism not supported for 23S analysis" Organism outside the 5 supported species. --skip-rrna23s still runs AMRFinderPlus.
Pileup TSV is empty BAM did not align reads to the 23S reference — check sample.23S.map.log and read quality.

11 · Publishing on GitHub & bioconda

GitHub

gh repo create sintetico87/linezolid-amr --public --source=. --push
git tag v0.1.0
git push --tags

Bioconda

  1. Fork bioconda/bioconda-recipes on GitHub.
  2. Copy conda-recipe/meta.yaml from this repo to recipes/linezolid-amr/meta.yaml in your fork.
  3. Replace the source: block with the release tarball: 
    source:
  url: https://github.com/sintetico87/linezolid-amr/archive/refs/tags/v0.1.0.tar.gz
  sha256: <sha256 of the tarball>
  4. Compute the sha256: 
    curl -sL https://github.com/sintetico87/linezolid-amr/archive/refs/tags/v0.1.0.tar.gz \
  | shasum -a 256
  5. Open a pull request to bioconda/bioconda-recipes targeting the master branch. Bioconda CI builds, tests, and gates the merge. Once merged, the package becomes installable via conda install -c bioconda linezolid-amr.
Pre-flight check. Build the recipe locally before opening the PR — saves a round-trip with the bioconda CI: 
conda install -c conda-forge boa conda-build
conda-build conda-recipe/
